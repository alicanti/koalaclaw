# COGNITIVE INFRASTRUCTURE PROTOCOL
Version 1.0

This workspace operates with persistent cognition.

If it is not written, it does not exist.
If it is not reviewed, it does not compound.
If it is not structured, it cannot scale.

---

## DIRECTORY STRUCTURE

```
/state/mind/
    /logs/
        YYYY-MM-DD.md
    PROFILE.md
    PROJECTS.md
    DECISIONS.md
    ERRORS.md
```

All files are mandatory.
All sessions begin by loading them.

---

## SESSION BOOT SEQUENCE (NON-NEGOTIABLE)

Before responding to anything:

1. Read PROFILE.md
2. Read PROJECTS.md
3. Read DECISIONS.md
4. Read ERRORS.md
5. Read today's log file (or create it)

No execution before context.
No advice without state awareness.

---

## FILE SPECIFICATIONS

### PROFILE.md
Your identity, role, expertise, and behavioral guidelines.
Updated when your role evolves or new capabilities are added.

### PROJECTS.md
Active projects, their status, and next steps.
Updated after every significant project change.

### DECISIONS.md
Important decisions made, with reasoning and date.
Append-only. Never delete past decisions.

### ERRORS.md
Mistakes made, lessons learned, and prevention strategies.
Append-only. Review before making similar decisions.

### logs/YYYY-MM-DD.md
Daily activity log. Created fresh each day.
Contains: tasks completed, conversations had, decisions made, issues encountered.

---

## UPDATE RULES

- After every significant interaction, update relevant files.
- PROJECTS.md: update status, add new projects, mark completed ones.
- DECISIONS.md: append new decisions with date and reasoning.
- ERRORS.md: append mistakes immediately when they happen.
- Daily log: append a summary entry after each meaningful task.
